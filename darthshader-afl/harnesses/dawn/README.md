Building the fuzzing target `tint_afl_all_fuzzer` can be automated using the `build.sh` script, or performed manually.

### Automated Build

You can use the `build.sh` script to automate the process.

```bash
# Show usage
./build.sh -h

# Build using default paths (clones dawn to ./dawn and builds in ./dawn/out/build)
# Assumes afl-clang-fast is in your PATH
./build.sh

# Build specifying AFL path and custom directories
./build.sh -a /path/to/afl/bin -s /path/to/dawn_source -b /path/to/dawn_build

# Build using AFL++ Docker image
./build.sh -d

# Build using Docker with sudo
./build.sh -d -u
```

### Testing the Build

You can run the test script to verify that the build process works and the resulting binary can execute:

```bash
# Run tests (assumes afl-clang-fast is in your PATH)
./build_test.sh

# Run tests specifying AFL path
./build_test.sh -a /path/to/afl/bin

# Run tests using Docker
./build_test.sh -d

# Run tests using Docker with sudo
./build_test.sh -d -u

# Run tests and keep the build directory (for debugging or incremental builds)
./build_test.sh -d -k

```

The test script will build Dawn in a temporary directory and verify that `tint_afl_all_fuzzer` runs and responds as expected.

### Manual Build


If you prefer to build manually, the process works as follows:

```bash
git clone https://dawn.googlesource.com/dawn
cd dawn
git checkout 3de0f00
cp scripts/standalone.gclient .gclient
gclient sync # this might trigger an error but should be fine anyways
git apply /path/to/darthshader/darthshader-afl/harnesses/dawn/patches/dawn_3de0f00.diff

mkdir -p out/build
cd out/build
CC=/path/to/afl-clang-fast CXX=/path/to/afl-clang-fast++ cmake -GNinja -DTINT_BUILD_AFL_FUZZER=ON -DDAWN_ENABLE_ASAN=ON -DTINT_BUILD_MSL_WRITER=ON -DTINT_BUILD_SPV_WRITER=ON -DTINT_BUILD_HLSL_WRITER=ON -DCMAKE_BUILD_TYPE=Release -DCMAKE_CXX_COMPILER_AR=/usr/bin/llvm-ar-18 -DCMAKE_CXX_COMPILER_RANLIB=/usr/bin/llvm-ranlib-18 -DCMAKE_C_COMPILER_AR=/usr/bin/llvm-ar-18 -DCMAKE_C_COMPILER_RANLIB=/usr/bin/llvm-ranlib-18 -DCMAKE_CXX_FLAGS="-fuse-ld=lld" -DCMAKE_C_FLAGS="-fuse-ld=lld" ../..
ninja tint_afl_all_fuzzer
```

Now run the fuzzer, e.g.:
```bash
darthshader-afl tint_afl_all_fuzzer -o test_corpus
```

Note that the harness automatically attempts to load the dxcompiler shared library.
You probably need to set `LD_LIBRARY_PATH` to the folder containing `libdxcompiler.so`
If the library cannot be found/loaded, the harness will test tint only.
If the library is found, the shaders generated by tint will be compiled with dxcompiler.
